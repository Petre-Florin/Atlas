# Vision

## What this is

Atlas is where I track the parts of my life I want to be intentional about.
It's built as **Strands** — independent modules that share one design language
but never share data models or logic. That's a deliberate architecture
choice, not a feature list: it means adding something new is additive, and
removing something is deleting one folder, not untangling a monolith.

## Principles

These are filters for whether to build something, not promises about what
will exist:

- **A strand earns its place through use, not through the plan.** Goals,
  Habits, and Journal are in v1 because they're the daily loop. Nothing else
  gets added until v1 has actually been used daily for a few weeks.
- **Modular, not maximal.** New strands must not touch existing ones' data or
  logic. If a "new strand" requires refactoring an old one, it's not a new
  strand — it's a rewrite, and that's a much bigger decision.
- **Mine, not shared.** Single user, no accounts to manage, no growth loop,
  no one else's feedback to chase.
- **Track progress, not perform it.** This isn't for anyone else to see. No
  feature should exist because it would look good on a CV rather than
  because I'd actually use it.

## Not a roadmap

The original brief listed Career, Finance, Learning, Knowledge, Networking,
Health, and an AI Mentor as future strands. None of that is committed to.
This document isn't the place to plan them — when one is actually needed,
it gets its own scoping conversation, the same way Goals/Habits/Journal did.
The only thing carried forward from that list is the shape: whatever gets
built next follows the Strand pattern in the README.
